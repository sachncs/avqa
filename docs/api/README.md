# AVQA API Reference

This directory contains the autodoc stubs for the AVQA public API. The
actual API documentation is generated by Sphinx from the docstrings in
`src/avqa/`.

## Modules

- `avqa` ‚Äî top-level package (`AVQAttention`, `AVQConfig`, etc.).
- `avqa.config` ‚Äî immutable configuration dataclasses.
- `avqa.attention` ‚Äî online softmax state + correction operator.
- `avqa.codebook` ‚Äî hierarchical codebook with mean constraint.
- `avqa.quantizer` ‚Äî vector quantizer (Euclidean, hierarchical).
- `avqa.routing` ‚Äî importance + selectors (TopP, Threshold, Budget).
- `avqa.merge` ‚Äî merge strategies (Probability, Weighted, Logit, Normalized).
- `avqa.refinement` ‚Äî adaptive refinement orchestrator.
- `avqa.backend` ‚Äî backend abstraction (Torch, Triton).
- `avqa.cache` ‚Äî KV cache (InMemory, Paged).
- `avqa.scheduler` ‚Äî refinement scheduler (Default, Adaptive).
- `avqa.functional` ‚Äî stateless functional API.
- `avqa.profiling` ‚Äî profiler.
- `avqa.visualization` ‚Äî visualizer (JSON-only).
- `avqa.integrations` ‚Äî HF, vLLM, FlashAttention, xFormers adapters.

## Autodoc Configuration

```python
project = "AVQA"
extensions = ["sphinx.ext.autodoc", "sphinx.ext.napoleon"]
napoleon_google_docstring = True
autodoc_default_options = {"members": True, "show-inheritance": True}
```

Run `sphinx-build docs/api docs/_build` from the repository root to
generate the HTML documentation.
