# AVQA API Reference

This directory contains the autodoc stubs for the AVQA public API. The
actual API documentation is generated by Sphinx from the docstrings in
`src/avqa/`.

## Modules

- `avqa` — top-level package (`AVQAttention`, `AVQConfig`, etc.).
- `avqa.config` — immutable configuration dataclasses.
- `avqa.attention` — online softmax state + correction operator.
- `avqa.codebook` — hierarchical codebook with mean constraint.
- `avqa.quantizer` — vector quantizer (Euclidean, hierarchical).
- `avqa.routing` — importance + selectors (TopP, Threshold, Budget).
- `avqa.merge` — merge strategies (Probability, Weighted, Logit, Normalized).
- `avqa.refinement` — adaptive refinement orchestrator.
- `avqa.backend` — backend abstraction (Torch).
- `avqa.cache` — KV cache (InMemory, Paged).
- `avqa.scheduler` — refinement scheduler (Default, Adaptive).
- `avqa.hopfield` — HVAQ temperature schedules.
- `avqa.online_adaptation` — BCAR codebook adaptation.
- `avqa.functional` — stateless functional API.
- `avqa.profiling` — profiler.
- `avqa.visualization` — visualizer (JSON-only).

## Autodoc Configuration

```python
project = "AVQA"
extensions = ["sphinx.ext.autodoc", "sphinx.ext.napoleon"]
napoleon_google_docstring = True
autodoc_default_options = {"members": True, "show-inheritance": True}
```

Run `sphinx-build docs/api docs/_build` from the repository root to
generate the HTML documentation.
